Webstore Project

Project Overview

This repository contains a small demo web store application — a proof-of-concept (PoC) that showcases a product catalog, a checkout flow and a backoffice demo for order management. The project demonstrates integrating PayPal payments using the Payper Java SDK (payper-orders-v2 and payper-payments-v2).

Key Features

  • Product catalog with dynamic product cards
  • Checkout process with delivery address and payment method selection (PayPal or credit card)
  • Integration with PayPal using the Payper Java SDK
  • Backoffice demo to view orders, view history and capture authorized payments
  • REST API endpoints documented with OpenAPI/Swagger

Technology Stack

  • Java 21 - Main programming language
  • Spring Boot 3.5.6 - Backend framework for REST APIs and web application
  • Payper Java SDK - https://github.com/eealba/payper (payper-orders-v2 and payper-payments-v2)
  • MapStruct - Java bean mapping
  • Lombok - Boilerplate reduction
  • Springdoc OpenAPI / Swagger - API documentation and UI
  • Jackson - JSON serialization/deserialization
  • Bootstrap 5 - Responsive frontend styling

Requirements

You need a PayPal API credential (Client ID and Client Secret). See PayPal API Authentication for details.

  1. Go to the PayPal Developer Dashboard and log in or sign up.
  2. Select Apps & Credentials.
  3. New accounts come with a Default Application in the REST API apps section. To create a new project, select Create App.
  4. Copy the Client ID and Client Secret for your app.
  5. Create the file credentials.properties in ~/.payper/ as required by the LocalSystemPropertyConfig class, with the following content:
    PAYPAL-CLIENT-ID=<clientID>
    PAYPAL-CLIENT-SECRET=<secret>

How to Use

  1. Start the application from the project root: 
    ./mvnw spring-boot:run
  2. Open the UI in your browser:
    • Webstore (customer-facing): /webstore.html
    • Backoffice demo: /backoffice.html
    • API docs (Swagger / OpenAPI UI): /swagger-ui/index.html
  3. Use the Backoffice demo to fetch orders and order history, and to capture authorized payments.

Important API endpoints (examples)

  • GET /api/catalog — retrieve the product catalog
  • POST /api/checkout/orders — submit a checkout order
  • POST /api/checkout/orders/{referenceId}/capture — capture an order payment
  • POST /api/checkout/orders/{referenceId}/authorize — authorize an order payment
  • GET /api/backoffice/orders — list current orders
  • GET /api/backoffice/orders/history — list orders history (most recent first)

Note: This project is for demonstration purposes and showcases integration with the Payper SDK in a modern Java/Spring Boot environment.
